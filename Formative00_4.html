Basic Digital Technologies-Part 4: Dev Environment + Local AI

200 points

Learning Objectives

In business, your laptop is your primary work tool. In this course you will repeatedly use a standard development environment to work with files, data, APIs, and local AI models.

This part introduces a simple, repeatable setup:

  • PowerShell 7 for reliable commands and scripts
  • Miniforge (Conda) to create a clean, reusable Python environment
  • JupyterLab to run notebooks for labs and system checks
  • Ollama to run a small local LLM and call it through an API

After completing this assignment you will be able to:

  1. Install and verify PowerShell 7 and run basic diagnostic commands.
  2. Create a standard working folder structure in C:\Users\<StarID>\Temp and C:\Users\<StarID>\html.
  3. Install Miniforge and create a Python environment from a .yml file.
  4. Start JupyterLab, open a notebook, and run a full “system check”.
  5. Install Ollama, pull a starter model, and make a simple API call to your local LLM.

Conda Quick Reference Commands:

  • conda info — Show your Conda setup + active environment
  • conda env list — List all Conda environments (active one marked with *)
  • conda activate <ENV_NAME> — Switch to an environment (example: conda activate wsu_mis_base)
  • conda list — List packages installed in the active environment
  • where python — Confirm which Python executable is being used (path)
  • python -c "import sys; print(sys.executable)" — Print the exact Python path for the active environment

Important notes before you begin

  • Do NOT put Conda environments in OneDrive. (Use local folders indicated below)
  • For web demos and local servers, we will use: C:\Users\<StarID>\html and (in some courses) C:\Users\<StarID>\inetpub.
  • If something fails, take a screenshot of the error and write 1–2 sentences explaining what you tried.

New Exercise 12 — Install Miniforge + Create the Standard Python Environment

Goal: Install Miniforge, confirm it’s working, and create the course Python environment from a .yml file.

  1. Download the Miniforge installer (Windows x86_64) from Professor Paulson's ReadOnly OneDrive folder, available in D2L.
  2. Run the installer using the default location. (Do not install into OneDrive.)
  3. Open Miniforge Prompt (Start Menu).
  4. Verify Miniforge and Python are accessible: 
    conda --version

  5. Verify environment and executable paths to confirm you are using the Miniforge-managed Python: 
    conda env list
where python
python -c "import sys; print(sys.executable)"
  6. Install mamba into the base environment (faster installs): 
    conda install -n base mamba -y

  7. Create a working folder for this semester (use PowerShell 7): 
    New-Item -ItemType Directory -Force C:\Users\$env:USERNAME\Temp\WSU_MIS\env
New-Item -ItemType Directory -Force C:\Users\$env:USERNAME\Temp\WSU_MIS\notebooks
New-Item -ItemType Directory -Force C:\Users\$env:USERNAME\html\mis_api_demo
  8. Download the provided environment file wsu_mis_base.yml to:

    C:\Users\<StarID>\Temp\WSU_MIS\env\wsu_mis_base.yml

  9. In Miniforge Prompt, change into the folder and create the environment: 
    cd C:\Users\%USERNAME%\Temp\WSU_MIS\env
mamba env create -f wsu_mis_base.yml

    Required: Create the AI environment too. We keep AI-related packages separate so the base environment stays fast and stable. 

    mamba env create -f wsu_mis_ai.yml

    Required: Register both environments as Jupyter kernels so you can pick the correct kernel inside JupyterLab. 

    conda activate wsu_mis_base
python -m ipykernel install --user --name wsu_mis_base --display-name "WSU MIS Base (wsu_mis_base)"

conda activate wsu_mis_ai
python -m ipykernel install --user --name wsu_mis_ai --display-name "WSU MIS AI (wsu_mis_ai)"
  10. Activate the environment and verify: 
    conda activate wsu_mis_base
python --version
conda env list

Exercise 14 — Install Ollama + Call Your Local LLM via API

Goal: Install Ollama, pull a small model, and make a basic API call.

  1. Install Ollama for Windows (x64 is 1.2GB) from this link and accept adding Ollama to PATH.
  2. Open PowerShell 7 and verify Ollama:
ollama --version ollama list
  1. Pull the starter model (small enough for i5 / 16GB):
ollama pull llama3.2:1b

Optional (higher quality, slower on CPU): ollama pull llama3.2:3b

  1. Run a quick local chat test:
ollama run llama3.2:1b
# Then type: Explain what an API is in exactly 2 sentences for a business student.
  1. Make a direct API call (this proves your laptop can call a local LLM like an app would):
$body = @{ model="llama3.2:1b"; prompt="Write a 3-bullet checklist for a student submitting an assignment." } | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:11434/api/generate" -Method Post -Body $body -ContentType "application/json"

Common Conda Commands

  • conda --version — Show Conda version
  • conda info — Summary of your Conda setup (active env, base env, channels, platform)
  • conda config --show — Show your Conda configuration (channels, settings)
  • conda config --show-sources — Show where config values are coming from (.condarc locations)
  • conda env list (or conda info --envs) — List all Conda environments and the active one
  • conda list — List packages installed in the active environment
  • conda list -n <ENV_NAME> — List packages installed in a specific environment
  • conda list --explicit — Export the exact package specs for the active environment
  • where conda (Windows) — Show which conda.exe is being used (path)
  • where python (Windows) — Show which python.exe is being used (path)
  • python -c "import sys; print(sys.executable)" — Print the exact Python executable path
  • conda env export --from-history — Export only the packages you explicitly installed (cleaner environment file)
  • conda env export — Export the full environment (often includes many dependencies)
  • conda search <PACKAGE> — Search for a package in configured channels
  • conda search -c conda-forge <PACKAGE> — Search a specific channel (example: conda-forge)
  • conda clean --all — Clean caches (can reclaim disk space; use with care)
  • conda list --revisions — Show install history (revisions) for the current environment
  • conda install --revision <N> — Roll back environment to a previous revision number
  • pip --version — Show pip version (inside the active environment)
  • pip list — List pip-installed packages (inside the active environment)
  • Models (if you use Ollama): ollama list — List local Ollama models
  • Models (if you use LM Studio CLI, if installed): lms --help — Show LM Studio CLI commands
