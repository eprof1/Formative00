Formative00 Part 5 — Build Your Lifelong Student Notebook (OneNote + NotebookLM + Open Notebook)

AI is powerful, but it is not magic. In this course you will learn to: collect evidence (screenshots), produce artifacts (your completed work), and reflect (what you learned and why it matters).

In Part 5 we add a “lifelong notebook” system that you can use across every course and later in your career. This semester OneNote is required, and you will also use Google NotebookLM and Open Notebook so you can compare approaches and avoid vendor lock-in.

After completing this assignment you will be able to:

  1. Create a single notebook that can scale across all courses using a consistent naming system
  2. Create and save a reusable OneNote page template for Assignment Submission Artifacts
  3. Use NotebookLM for source-grounded study (citations to your uploaded material)
  4. Install Open Notebook (self-hosted, Docker) and connect it to a local Ollama model provider
  5. Explain how model choice + knowledge source choice affects quality, speed, cost, and privacy

Exercise 5 — Open Notebook (Docker) + Ollama (Required)

Goal

Install Open Notebook using Docker and connect it to Ollama. This gives you a vendor-neutral notebook that can run with local models.

Required before you start

  • Ollama installed and running on your laptop (http://localhost:11434)
  • Docker Desktop installed (WSL2 backend)

Steps (Docker)

  1. Create a folder: C:\AI\open-notebook
  2. In PowerShell 7, verify Docker + WSL2:
docker --version
wsl --status
  1. Follow the instructor-provided docker compose file for Open Notebook, then start it:
cd C:\AI\open-notebook
docker compose up -d

Note: If Open Notebook runs in Docker, it may need to reach Ollama using http://host.docker.internal:11434 (not localhost inside the container).

Connect Open Notebook to Ollama

  1. Open the Open Notebook web UI in your browser.
  2. Go to Settings → Models (or similar).
  3. Create at least:
    • One Language model (example: a small Ollama model for speed)
    • One Embedding model (example: an Ollama embedding model)
  4. Set your defaults so Open Notebook uses your selected models.

Exercise 6 — Compare the three tools (Required)

Goal

Learn the key “AI is not magic” skill: you must choose (1) the model and (2) the knowledge source intentionally.

Steps

  1. Pick one small course topic (example: “What is an API?” or “What is RAG?”).
  2. Run the same prompt in all three tools:
    • OneNote (you write your own notes)
    • NotebookLM (must cite your uploaded sources)
    • Open Notebook (Ollama-based model)
  3. In OneNote, write a short comparison (6–10 lines):
    • Which was fastest?
    • Which was most trustworthy?
    • Which is easiest to export/share?
    • Which feels “locked in” to a vendor?
